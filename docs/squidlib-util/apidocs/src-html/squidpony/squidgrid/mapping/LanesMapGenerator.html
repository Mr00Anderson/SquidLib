008/**
009 * Generate dungeons with between 1 and 3 primary "lanes" going from the upper left "base" to the bottom right "base"
010 * (and vice versa, since this is symmetrical). Also fills the area not covered by lanes with "jungle" (random, but
011 * symmetrical, room or cave connections). Dungeons are produced by MixedGenerator, like those SerpentMapGenerator
012 * makes, but include the wide lanes going from corner to corner. You can call different methods like putCaveCarvers(),
013 * putBoxRoomCarvers(), putWalledRoundRoomCarvers(), etc. to affect the "jungle", which defaults to caves unless one or
014 * more of the putXXXCarvers methods was called. The lanes are always 5 floor cells wide, measured 8-way. This supports
015 * the getEnvironment() method, which can be used in conjunction with RoomFinder to find where separate room, corridor,
016 * and cave areas have been placed.
017 * <br>
018 * A preview can be seen here https://gist.github.com/tommyettinger/4f57cff23eead11b17bf , with dungeons created with
019 * one, two, and three lanes, and only using box-shaped rooms for "jungle." Currently, the two-lane dungeon seems to be
020 * ideal for maps that aren't incredibly large; the samples are 80x80, but larger maps may have better jungle layout
021 * with three lanes than those three-lane maps can manage on smaller sizes. Another potential advantage of the two-lane
022 * approach is that it can be used to generate a "ring" of wide paths around a central "core" of jungle, which wasn't
023 * originally intended as a use of this generator but could be very useful for games that, for instance, want guards
024 * patrolling an obvious ring, while the player, monsters, and/or other prisoners start in the jungle.
025 * Created by Tommy Ettinger on 10/24/2015.
026 */
027public class LanesMapGenerator implements IDungeonGenerator {
028    protected SymmetryDungeonGenerator mix;
029    protected int[] columns, rows;
030    protected IRNG random;
031    protected int lanes;

036     * non-lane portion of the dungeon will look like, putCaveCarvers(), putBoxRoomCarvers(), and putRoundRoomCarvers(),
037     * defaulting to only caves if none are called. You call generate() after adding carvers, which returns a char[][]
038     * for a map.

169    /**
170     * Changes the number of "carvers" that will create caves from one room to the next. If count is 0 or less, no caves
171     * will be made. If count is at least 1, caves are possible, and higher numbers relative to the other carvers make
172     * caves more likely. Carvers are shuffled when used, then repeat if exhausted during generation. Since typically
173     * about 30-40 rooms are carved, large totals for carver count aren't really needed; aiming for a total of 10
174     * between the count of putCaveCarvers(), putBoxRoomCarvers(), and putRoundRoomCarvers() is reasonable.
175     * @see MixedGenerator
176     * @param count the number of carvers making caves between rooms; only matters in relation to other carvers
177     */
178    public void putCaveCarvers(int count)
179    {
180        mix.putCaveCarvers(count);
181    }
182    /**
183     * Changes the number of "carvers" that will create right-angle corridors from one room to the next, create rooms
184     * with a random size in a box shape at the start and end, and a small room at the corner if there is one. If count
185     * is 0 or less, no box-shaped rooms will be made. If count is at least 1, box-shaped rooms are possible, and higher
186     * numbers relative to the other carvers make box-shaped rooms more likely. Carvers are shuffled when used, then
187     * repeat if exhausted during generation. Since typically about 30-40 rooms are carved, large totals for carver
188     * count aren't really needed; aiming for a total of 10 between the count of putCaveCarvers(), putBoxRoomCarvers(),
189     * and putRoundRoomCarvers() is reasonable.
190     * @see MixedGenerator
191     * @param count the number of carvers making box-shaped rooms and corridors between them; only matters in relation
192     *              to other carvers
193     */

195    {
196        mix.putBoxRoomCarvers(count);
197    }

250    /**
251     * This generates a new map by stretching a 16x16 grid of potential rooms to fit the width and height passed to the
252     * constructor, randomly expanding columns and rows before contracting the whole to fit perfectly. This uses the
253     * Moore Curve, a space-filling curve that loops around on itself, to guarantee that the rooms will always have a
254     * long path through the dungeon that, if followed completely, will take you back to your starting room. Some small
255     * branches are possible, and large rooms may merge with other rooms nearby. This uses MixedGenerator.
256     * @see MixedGenerator
257     * @return a char[][] where '#' is a wall and '.' is a floor or corridor; x first y second
258     */
259    public char[][] generate()
260    {
261        return mix.generate();
262    }

264    /**
265     * Gets a 2D array of int constants, each representing a type of environment corresponding to a static field of
266     * MixedGenerator. This array will have the same size as the last char 2D array prduced by generate(), and the value
267     * of this method if called before generate() is undefined, but probably will be a 2D array of all 0 (UNTOUCHED).
268     * <ul>
269     *     <li>MixedGenerator.UNTOUCHED, equal to 0, is used for any cells that aren't near a floor.</li>
270     *     <li>MixedGenerator.ROOM_FLOOR, equal to 1, is used for floor cells inside wide room areas.</li>
271     *     <li>MixedGenerator.ROOM_WALL, equal to 2, is used for wall cells around wide room areas.</li>
272     *     <li>MixedGenerator.CAVE_FLOOR, equal to 3, is used for floor cells inside rough cave areas.</li>
273     *     <li>MixedGenerator.CAVE_WALL, equal to 4, is used for wall cells around rough cave areas.</li>
274     *     <li>MixedGenerator.CORRIDOR_FLOOR, equal to 5, is used for floor cells inside narrow corridor areas.</li>
275     *     <li>MixedGenerator.CORRIDOR_WALL, equal to 6, is used for wall cells around narrow corridor areas.</li>
276     * </ul>
277     * @return a 2D int array where each element is an environment type constant in MixedGenerator
278     */
279    public int[][] getEnvironment()
280    {
281        return mix.getEnvironment();
282    }

284    public char[][] getDungeon() {
285        return mix.getDungeon();
286    }
