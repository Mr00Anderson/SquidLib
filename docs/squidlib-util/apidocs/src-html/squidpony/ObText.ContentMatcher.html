015/**
016 * A simple format parser for String-based configuration or data files where JSON is overkill.
017 * Supports only one type, String, but allows each String to have arbitrary nested levels of
018 * String children as if in sub-lists. You can interpret the Strings however you want, and
019 * quoting each String isn't necessary if they are just one word ("bare words" are allowed).
020 * This stores its items in an inner class, {@link ObTextEntry}, which only has a "primary"
021 * String and may have a List of "associated" ObTextEntry values, each of which must have
022 * their own primary String and which may have their own associated List.
023 * <br>
024 * You can use this like any other List, though it will be contain ObTextEntry objects instead
025 * of Strings directly. This allows you to control whether you want to iterate through a
026 * particular primary String's associated entries, if there are any, or to skip over them and
027 * go to the next String in the current List.
028 * <br>
029 * This implements List of String and is modifiable; it extends the List interface with
030 * {@link #parse(CharSequence)} to read a String in the following format, as well as with some
031 * overloads of add() to add just a String with no associated values. A quirk of how this
032 * implements List is that it only considers the top-level Strings to be part of the List for
033 * length and for {@link #contains(Object)}, and will ignore child strings unless you access
034 * them via the {@link ObTextEntry#associated} List on an entry that has associated entries.
035 * <br>
036 * Format example:
037 * <pre>
038 * hello world
039 * 'how are you today?' [just great thanks]
040 * hooray!
041 *
042 * complexity?
043 * [it is possible [yes this is a good example]
044 * 'escapes like \[\'\] all work'
045 * "you can use double or single quotes to allow spaces and brackets in one string"
046 * ]
047 *
048 * comments are allowed // like this
049 * comments can have different forms # like this
050 * // block comments like in c are allowed
051 * / * but because this example is in javadoc, this example is not actually a comment * /
052 * // remove the spaces between each slash and asterisk to make the last line a comment.
053 * /[delimit/or block comments with delimiters/delimit]/
054 *
055 * '''
056 * raw strings (heredocs) look like this normally.
057 *     they permit characters without escapes, ]][][[ \/\/\ ,
058 *     except for triple quotes.
059 *     they keep newlines and indentation intact,
060 * except for up to one newline ignored adjacent to each triple quote.
061 * '''
062 *
063 * [[different[
064 * if you may need triple quotes
065 *     in the raw string, use a different syntax that allows delimiters.
066 * here, the delimiter is '''different''', just to be different.]different]]
067 * </pre>
068 * <br>
069 * Inspired strongly by STOB, https://github.com/igagis/stob-java and https://github.com/igagis/stob , but
070 * no code is shared and the format is slightly different. The main differences are:
071 * <ul>
072 * <li>We use square brackets in place of STOB's curly braces to mark children associated with a string.<li>
073 * <li>ObText supports nested block comments using the syntax {@code /[delimiter/contents/delimiter]/} where
074 * delimiter may be empty but must match on both sides, and contents is the body of the comment.</li>
075 * <li>ObText uses Python-like "heredoc" syntax for raw strings surrounded by triple-apostrophes '''like so'''
076 * with optional initial and final newlines in the raw string ignored. An alternate raw string
077 * syntax is present that allows delimiters, using the syntax {@code [[delimiter[contents]delimiter]]}, where 
078 * again delimiter may be empty and contents is the body of the raw string.</li>
079 * <ul>
080 */

235    /**
236     * Parses the given text (a String or other CharSequence) and appends it into this ObText.
237     * @param text a CharSequence (such as a String) using ObText formatting, as described in this class' JavaDocs
238     * @return this ObText object after appending the parsed text, for chaining
239     */
240    public ObText parse(CharSequence text)
241    {
242        m.setTarget(text);
243        ObTextEntry current = null;
244        List<ObTextEntry> ls = this;
245        IntVLA nesting = new IntVLA(4);
246        nesting.add(-1);
247        int depth = 0;
248        while (m.find()) {
249            if (m.isCaptured(stringId)) {
250                ls.add(current = new ObTextEntry(m.group(stringId)));
251                nesting.incr(depth, 1);
252            }
253            else if(m.isCaptured(openId))
254            {
255                if(current == null) throw new UnsupportedOperationException("ObText entries can't have associated items without a primary String.");
256                nesting.add(-1);
257                ls = current.openAssociated();
258                depth++;
259            }
260            else if(m.isCaptured(closeId))
261            {
262                if(nesting.size <= 1) throw new UnsupportedOperationException("Associated item sequences in ObText can't end more times than they start.");
263                nesting.pop();
264                depth--;
265                ls = this;
266                for (int i = 0; i < depth; i++) {
267                    ls = ls.get(nesting.get(i)).associated;
268                }
269            }
270        }
271        return this;
272    }

312    // Used to generate randomized delimiters using up to 9 non-English letters.
313    // call while assigning your state with randomChars(state += 0x9E3779B97F4A7C15L, myChars)
314    // that assumes you have a 9-element char[] called myChars
315    // as long as z/state is deterministic (i.e. based on a hash), this should be too
316    private static void randomChars(long z, char[] mut)
317    {
318        z = (z ^ (z >>> 30)) * 0xBF58476D1CE4E5B9L;
319        z = (z ^ (z >>> 27)) * 0x94D049BB133111EBL;
320        z ^= (z >>> 31);
321
322        mut[0] = letters[(int)(128 + (z & 127))];
323        mut[1] = letters[(int)(128 + (z >>> 7 & 127))];
324        mut[2] = letters[(int)(128 + (z >>> 14 & 127))];
325        mut[3] = letters[(int)(128 + (z >>> 21 & 127))];
326        mut[4] = letters[(int)(128 + (z >>> 28 & 127))];
327        mut[5] = letters[(int)(128 + (z >>> 35 & 127))];
328        mut[6] = letters[(int)(128 + (z >>> 42 & 127))];
329        mut[7] = letters[(int)(128 + (z >>> 49 & 127))];
330        mut[8] = letters[(int)(128 + (z >>> 56 & 127))];
331    }

342    protected static void appendQuoted(StringBuilder sb, String text, Matcher bareFinder)
343    {
344        if(text == null || text.isEmpty()) {
345            sb.append("''");
346            return;
347        }
348        bareFinder.setTarget(text);
349        if(!bareFinder.find())
350            sb.append(text);
351        else
352        {
353            raw.setTarget(text);
354            if(raw.find()) {
355
356                if (text.contains("'''")) {
357                    long state = CrossHash.hash64(text);
358                    char[] myChars = new char[9];
359                    int count;
360                    do {
361                        randomChars(state += 0x9E3779B97F4A7C15L, myChars);
362                        count = StringKit.containsPart(text, myChars, "]", "]]");
363                    } while (count == 12);
364                    sb.append("[[").append(myChars, 0, count).append("[\n").append(text).append("\n]")
365                            .append(myChars, 0, count).append("]]");
366                } else {
367                    sb.append("'''\n").append(text).append("\n'''");
368                }
369            }
370            else if(!text.contains("'"))
371            {
372                sb.append('\'').append(text).append('\'');
373            }
374            else
375            {
376                if(text.contains("\""))
377                {
378                    if(text.contains("'''"))
379                    {
380                        long state = CrossHash.hash64(text);
381                        char[] myChars = new char[9];
382                        int count;
383                        do
384                        {
385                            randomChars(state += 0x9E3779B97F4A7C15L, myChars);
386                            count = StringKit.containsPart(text, myChars);
387                        }while(count == 9);
388                        sb.append("[[").append(myChars, 0, count).append("[\n").append(text).append("\n]")
389                                .append(myChars, 0, count).append("]]");
390                    }
391                    else
392                    {
393                        sb.append("'''\n").append(text).append("\n'''");
394                    }
395                }
396                else
397                {
398                    sb.append('"').append(text).append('"');
399                }
400            }
401        }
402    }

432    private static void iterate(StringBuilder sb, ArrayList<ObTextEntry> obt)
433    {
434        int len = obt.size();
435        ObTextEntry entry;
436        for (int i = 0; i < len; i++) {
437            appendQuotedObText(sb, (entry = obt.get(i)).primary);
438            sb.append('\n');
439            if(entry.hasAssociated())
440            {
441                sb.append("[\n");
442                iterate(sb, entry.associated);
443                sb.append("]\n");
444            }
445        }
446    }

449    {

451        ObTextEntry entry;
452        for (int i = 0; i < len; i++) {
453            buffer.add((entry = obt.get(i)).primary);
454            if(entry.hasAssociated())
455            {
456                iterate(buffer, entry.associated);
457            }
458        }

588     * Can be used to help reading sequences of Strings with ObText-style quotation marking their boundaries.
589     * This returns a {@link ContentMatcher} object that you must call setTarget on before using it.
590     * The argument(s) to setTarget should be the text that might contain quotes, heredoc-style quotes, or just bare
591     * words. Calling {@link ContentMatcher#find()} will try to find the next String, returning false if there's nothing
592     * left or returning true and advancing the search if a String was found. The String might be a special term in some
593     * cases, like "[" and "]" without quotes being syntax in ObText that don't contain usable Strings. That's why,
594     * after a String was found with find(), you should check {@link ContentMatcher#hasMatch()} to verify that a match
595     * was successful, and if that's true, then you can call {@link ContentMatcher#getMatch()} to get the un-quoted
596     * contents of the next String in the target.
597     * @return a {@link ContentMatcher} that must have one of its setTarget() methods called before it can be used
598     */
599    public static ContentMatcher makeMatcher()
600    {
601        return new ContentMatcher();
602    }
603    /**
604     * Can be used to help reading sequences of Strings with ObText-style quotation marking their boundaries.
605     * This returns a {@link ContentMatcher} object that is already configured to read from {@code text}.
606     * The {@code text} should contain Strings that may be surrounded by quotes, heredoc-style quotes, or just bare
607     * words. Calling {@link ContentMatcher#find()} will try to find the next String, returning false if there's nothing
608     * left or returning true and advancing the search if a String was found. The String might be a special term in some
609     * cases, like "[" and "]" without quotes being syntax in ObText that don't contain usable Strings. That's why,
610     * after a String was found with find(), you should check {@link ContentMatcher#hasMatch()} to verify that a match
611     * was successful, and if that's true, then you can call {@link ContentMatcher#getMatch()} to get the un-quoted
612     * contents of the next String in the target.
613     * @param text the target String that should probably have at least one sub-string that might be quoted
614     * @return a {@link ContentMatcher} that can be used immediately by calling {@link ContentMatcher#find()}
615     */
616    public static ContentMatcher makeMatcher(CharSequence text)
617    {
618        return new ContentMatcher(text);
619    }
620
621    /**
622     * Can be used to help reading sequences of Strings with ObText-style quotation marking their boundaries, but no
623     * comments (which allows some additional characters to be used in bare words, like '#').
624     * This returns a {@link ContentMatcher} object that is already configured to read from {@code text}.
625     * The {@code text} should contain Strings that may be surrounded by quotes, heredoc-style quotes, or just bare
626     * words. Calling {@link ContentMatcher#find()} will try to find the next String, returning false if there's nothing
627     * left or returning true and advancing the search if a String was found. Unlike the ContentMatcher produced by
628     * {@link #makeMatcher(CharSequence)}, you can call {@link ContentMatcher#getMatch()} after any successful call to
629     * {@link ContentMatcher#find()}, which will get the un-quoted contents of the next String in the target.
630     * @param text the target String that should probably have at least one sub-string that might be quoted
631     * @return a {@link ContentMatcher} that can be used immediately by calling {@link ContentMatcher#find()}
632     */
633    public static ContentMatcher makeMatcherNoComments(CharSequence text)
634    {
635        return new ContentMatcher(text, patternRelaxed);
636    }

638    public static class ContentMatcher extends Matcher {
639
640        /**
641         * Constructs a ContentMatcher that will need to have its target set with {@link #setTarget(CharSequence)} or
642         * one of its overloads. The target should contain multiple substrings that may have quotation around them; this
643         * class is meant to skip the quotation in ObText's style.
644         */
645        public ContentMatcher()
646        {
647            super(pattern);
648        }
649
650        /**
651         * Constructs a ContentMatcher that already has its target set to {@code text}.
652         * @param text the CharSequence, such as a String, to find possibly-quoted Strings in.
653         */
654        public ContentMatcher(CharSequence text)
655        {
656            super(pattern, text);
657        }
658        /**
659         * Constructs a ContentMatcher that already has its target set to {@code text} and uses an alternate Pattern.
660         */
661        ContentMatcher(CharSequence text, Pattern altPattern)
662        {
663            super(altPattern, text);
664        }
665
666
667        /**
668         * Supplies a text to search in/match with.
669         * Resets current search position to zero.
670         *
671         * @param text - a data
672         * @see Matcher#setTarget(Matcher, int)
673         * @see Matcher#setTarget(CharSequence, int, int)
674         * @see Matcher#setTarget(char[], int, int)
675         * @see Matcher#setTarget(Reader, int)
676         */
677        @Override
678        public void setTarget(CharSequence text) {
679            super.setTarget(text);
680        }
681
682        /**
683         * Supplies a text to search in/match with, as a part of String.
684         * Resets current search position to zero.
685         *
686         * @param text  - a data source
687         * @param start - where the target starts
688         * @param len   - how long is the target
689         * @see Matcher#setTarget(Matcher, int)
690         * @see Matcher#setTarget(CharSequence)
691         * @see Matcher#setTarget(char[], int, int)
692         * @see Matcher#setTarget(Reader, int)
693         */
694        @Override
695        public void setTarget(CharSequence text, int start, int len) {
696            super.setTarget(text, start, len);
697        }
698
699        /**
700         * Supplies a text to search in/match with, as a part of char array.
701         * Resets current search position to zero.
702         *
703         * @param text  - a data source
704         * @param start - where the target starts
705         * @param len   - how long is the target
706         * @see Matcher#setTarget(Matcher, int)
707         * @see Matcher#setTarget(CharSequence)
708         * @see Matcher#setTarget(CharSequence, int, int)
709         * @see Matcher#setTarget(Reader, int)
710         */
711        @Override
712        public void setTarget(char[] text, int start, int len) {
713            super.setTarget(text, start, len);
714        }
715
716        /**
717         * Returns true if {@link #find()} has returned true and the found text is a usable String (not some syntax).
718         * If this returns true, you can reasonably get a (possibly empty) String using {@link #getMatch()}.
719         * @return true if there is a usable String found that can be obtained with {@link #getMatch()}
720         */
721        public boolean hasMatch()
722        {
723            return isCaptured(stringId);
724        }
725
726        /**
727         * Returns the contents of the latest String successfully found with {@link #find()}, without quotation.
728         * You should typically call {@link #hasMatch()} even if find() has returned true, to ensure there is a valid
729         * String that can be acquired (this will return an empty String if hasMatch() returns false, but an empty
730         * String is also potentially a valid result in a successful match, so it should be distinguished).
731         * @return the contents of the latest String successfully found with {@link #find()}
732         */
733        public String getMatch()
734        {
735            return group(stringId);
736        }
737    }
