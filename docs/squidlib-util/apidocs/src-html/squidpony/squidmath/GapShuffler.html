011/**
012 * Meant to take a fixed-size set of items and produce a shuffled stream of them such that an element is never chosen in
013 * quick succession; that is, there should always be a gap between the same item's occurrences. This is an Iterable of
014 * T, not a Collection, because it can iterate without stopping, infinitely, unless you break out of a foreach loop that
015 * iterates through one of these, or call the iterator's next() method only a limited number of times. Collections have
016 * a size that can be checked, but Iterables can be infinite (and in this case, this one is).
017 * Created by Tommy Ettinger on 5/21/2016.
018 * @param <T> the type of items to iterate over; ideally, the items are unique
019 */
020public class GapShuffler<T> implements Iterator<T>, Iterable<T>, Serializable {
021    private static final long serialVersionUID = 1L;
022    public IRNG rng;
023    private ArrayList<T> elements;
024    private int index;
025    private GapShuffler() {}
026
027    public GapShuffler(T single)
028    {
029        rng = new RNG();
030        elements = new ArrayList<>(1);
031        elements.add(single);
032        index = 0;
033    }
034    /**
035     * Constructor that takes any Collection of T, shuffles it with an unseeded RNG, and can then iterate infinitely
036     * through mostly-random shuffles of the given collection. These shuffles are spaced so that a single element should
037     * always have a large amount of "gap" in order between one appearance and the next. It helps to keep the appearance
038     * of a gap if every item in elements is unique, but that is not necessary and does not affect how this works.
039     * @param elements a Collection of T that will not be modified
040     */
041    public GapShuffler(Collection<T> elements)
042    {
043        this(elements, new RNG(new LongPeriodRNG()));
044
045    }

059    /**
060     * Constructor that takes any Collection of T, shuffles it with the given RNG, and can then iterate infinitely
061     * through mostly-random shuffles of the given collection. These shuffles are spaced so that a single element should
062     * always have a large amount of "gap" in order between one appearance and the next. It helps to keep the appearance
063     * of a gap if every item in items is unique, but that is not necessary and does not affect how this works. The
064     * rng parameter is copied so externally using it won't change the order this produces its values; the rng field is
065     * used whenever the iterator needs to re-shuffle the internal ordering of items. I suggest that the IRNG should
066     * use {@link LongPeriodRNG} as its RandomnessSource, since it is in general a good choice for shuffling, or
067     * {@link XoshiroStarPhi32RNG} for GWT or other 32-bit platforms, but the choice is unlikely to matter in practice.
068     * @param items a Collection of T that will not be modified
069     * @param rng an IRNG that can be pre-seeded; will be copied and not used directly
070     */
071    public GapShuffler(Collection<T> items, IRNG rng)
072    {
073        this.rng = rng.copy();
074        elements = this.rng.shuffle(items);
075        index = 0;
076    }

078    /**
079     * Constructor that takes any Collection of T, shuffles it with the given RNG, and can then iterate infinitely
080     * through mostly-random shuffles of the given collection. These shuffles are spaced so that a single element should
081     * always have a large amount of "gap" in order between one appearance and the next. It helps to keep the appearance
082     * of a gap if every item in items is unique, but that is not necessary and does not affect how this works. The
083     * rng parameter will be copied if {@code shareRNG} is true, otherwise the reference will be shared (which could
084     * make the results of this GapShuffler depend on outside code, though it will always maintain a gap between
085     * identical elements if the elements are unique). I suggest that the IRNG should use {@link LongPeriodRNG} as its
086     * RandomnessSource, since it is in general a good choice for shuffling, or {@link XoshiroStarPhi32RNG} for GWT or
087     * other 32-bit platforms, but the choice is unlikely to matter in practice. Any {@link IStatefulRNG} should work in
088     * most cases, like {@link GWTRNG}, {@link StatefulRNG}, or {@link SilkRNG}. If you encounter unusually-low-quality
089     * shuffles, try a different starting seed, and if that doesn't work, try SilkRNG (it has a different internal
090     * structure than the other types; it could yield better results at the very start).
091     * @param items a Collection of T that will not be modified
092     * @param rng an IRNG that can be pre-seeded; will be copied and not used directly
093     * @param shareRNG if false, {@code rng} will be copied and no reference will be kept; if true, {@code rng} will be shared with the outside code
094     */
095    public GapShuffler(Collection<T> items, IRNG rng, boolean shareRNG)
096    {
097        this.rng = shareRNG ? rng : rng.copy();
098        elements = this.rng.shuffle(items);
099        index = 0;
100    }

127     * Constructor that takes any Collection of T, shuffles it with the given RNG, and can then iterate infinitely
128     * through mostly-random shuffles of the given collection. These shuffles are spaced so that a single element should
129     * always have a large amount of "gap" in order between one appearance and the next. It helps to keep the appearance
130     * of a gap if every item in items is unique, but that is not necessary and does not affect how this works. The
131     * rng parameter is copied so externally using it won't change the order this produces its values; the rng field is
132     * used whenever the iterator needs to re-shuffle the internal ordering of items. I suggest that the IRNG should
133     * use {@link LongPeriodRNG} as its RandomnessSource, since it is in general a good choice for shuffling, or
134     * {@link XoshiroStarPhi32RNG} for GWT or other 32-bit platforms, but the choice is unlikely to matter in practice.
135     * @param items a Collection of T that will not be modified
136     * @param rng an IRNG that can be pre-seeded; will be copied and not used directly
137     */
138    public GapShuffler(T[] items, IRNG rng)
139    {
140        this.rng = rng.copy();
141        elements = Maker.makeList(items);
142        this.rng.shuffleInPlace(elements);
143        index = 0;
144    }

146    /**
147     * Constructor that takes any Collection of T, shuffles it with the given RNG, and can then iterate infinitely
148     * through mostly-random shuffles of the given collection. These shuffles are spaced so that a single element should
149     * always have at least one "gap" element between one appearance and the next. It helps to keep the appearance
150     * of a gap if every item in items is unique, but that is not necessary and does not affect how this works. The
151     * rng parameter will be copied if {@code shareRNG} is false, otherwise the reference will be shared (which could
152     * make the results of this GapShuffler depend on outside code, though it will always maintain a gap between
153     * identical elements if the elements are unique). I suggest that the IRNG should use {@link LongPeriodRNG} as its
154     * RandomnessSource, since it is in general a good choice for shuffling, or {@link XoshiroStarPhi32RNG} for GWT or
155     * other 32-bit platforms, but the choice is unlikely to matter in practice. Any {@link IStatefulRNG} should work in
156     * most cases, like {@link GWTRNG}, {@link StatefulRNG}, or {@link SilkRNG}. If you encounter unusually-low-quality
157     * shuffles, try a different starting seed, and if that doesn't work, try SilkRNG (it has a different internal
158     * structure than the other types; it could yield better results at the very start).
159     * @param items a Collection of T that will not be modified
160     * @param rng an IRNG that can be pre-seeded; will be copied and not used directly
161     * @param shareRNG if false, {@code rng} will be copied and no reference will be kept; if true, {@code rng} will be shared with the outside code
162     */
163    public GapShuffler(T[] items, IRNG rng, boolean shareRNG)
164    {
165        this.rng = shareRNG ? rng : rng.copy();
166        elements = Maker.makeList(items);
167        this.rng.shuffleInPlace(elements);
168        index = 0;
169    }
170
171    /**
172     * Gets the next element of the infinite sequence of T this shuffles through. This class can be used as an
173     * Iterator or Iterable of type T.
174     * @return the next element in the infinite sequence
175     */
176    public T next() {
177        int size = elements.size();
178        if(size == 1)
179        {
180            return elements.get(0);
181        }
182        if(index >= size)
183        {
184            final int n = size - 1;
185            for (int i = n; i > 1; i--) {
186                Collections.swap(elements, rng.nextSignedInt(i), i - 1);
187            }
188            Collections.swap(elements, 1 + rng.nextSignedInt(n), n);
189            index = 0;
190        }
191        return elements.get(index++);
192    }
193    /**
194     * Returns {@code true} if the iteration has more elements.
195     * This is always the case for GapShuffler.
196     *
197     * @return {@code true} always
198     */
199    @Override
200    public boolean hasNext() {
201        return true;
202    }
203    /**
204     * Not supported.
205     *
206     * @throws UnsupportedOperationException always throws this exception
207     */
208    @Override
209    public void remove() {
210        throw new UnsupportedOperationException("remove() is not supported");
211    }

213    /**
214     * Returns an <b>infinite</b> iterator over elements of type {@code T}; the returned iterator is this object.
215     * You should be prepared to break out of any for loops that use this once you've gotten enough elements!
216     * The remove() method is not supported by this iterator and hasNext() will always return true.
217     *
218     * @return an infinite Iterator over elements of type T.
219     */
220    @Override
221    public Iterator<T> iterator() {
222        return this;
223    }

225    public IRNG getRNG() {
226        return rng;
227    }

230    /**
231     * Sets the IRNG this uses to shuffle the order of elements, always copying the given IRNG before using it. Always
232     * reshuffles the order, which may eliminate a gap that should have been present, so treat the sequence before and
233     * after like separate GapShuffler objects.
234     * @param rng an IRNG, such as {@link RNG}, {@link GWTRNG}, {@link StatefulRNG}, and so on; always copied
235     */
236    public void setRNG(IRNG rng) {
237        setRNG(rng, false);
238    }

240    /**
241     * Sets the IRNG this uses to shuffle the order of elements, optionally sharing a reference between outside code and
242     * the internal rng (when {@code shareRNG} is true). Always reshuffles the order, which may eliminate a gap that
243     * should have been present, so treat the sequence before and after like separate GapShuffler objects.
244     * @param rng an IRNG, such as {@link RNG}, {@link GWTRNG}, {@link StatefulRNG}, and so on
245     * @param shareRNG if false, {@code rng} will be copied and no reference will be kept; if true, {@code rng} will be shared with the outside code
246     */
247    public void setRNG(IRNG rng, boolean shareRNG) {
248        this.rng = shareRNG ? rng : rng.copy();
249        this.rng.shuffleInPlace(elements);
250    }
