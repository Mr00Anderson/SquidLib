010/**
011 * Like a kind of RNG, but fully deterministic in a way that depends on a "connected" double array.
012 * Intended as a way to produce similar values when small changes occur in the connections, while potentially producing
013 * larger changes when the changes are more significant (unlike an RNG or hashing function, which can and should produce
014 * very different output given even slightly different seeds/input). This might be useful to produce procedural story
015 * data that is similar when most of the connected inputs are similar, or for terrain generation/population. This can
016 * produce ints and doubles, and does not produce a different output unless its input is changed (usually by altering a
017 * shared reference to {@code connections}). Also implements the various {@link Noise} interfaces, which this doesn't
018 * do perfectly but is at least different (it may yield large spans of high or low results, which Simplex and Perlin
019 * noise cannot actually do).
020 * <br>
021 * Created by Tommy Ettinger on 5/18/2017.
022 */
